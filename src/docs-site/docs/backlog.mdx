## 1. Overview

<div class="csnCard">

The **Product Backlog** defines the main development tasks for the ChildSafeNet platform.

The backlog is structured into **five development phases**, each focusing on a
specific part of the system:

- Core system functionality
- AI detection and browser integration
- Admin tools and training pipelines
- Documentation and release preparation
- DevOps and security hardening

This structure ensures **clear milestones, modular development, and stable releases**.

</div>

---

## 2. Development Phases

<div class="csnCard">

| Phase   | Focus                              |

## 4. AI & Browser Integration (Phase 2)

<div class="csnSteps">

<div class="csnStep">

<div class="csnStepTop">
  <div class="csnStepNum">1</div>
  <div class="csnStepTag">AI</div>
</div>

<h3 class="csnStepTitle">Hybrid AI Detection Engine</h3>

<p class="csnStepDesc">

Machine learning system for detecting unsafe URLs.

Key tasks:

- Extract **1000+ handcrafted URL features**
- Build **TF-IDF NLP pipeline**
- Combine scores via aggregation logic
- Configure detection thresholds
- Serialize trained models

</p>

</div>

<div class="csnStep">

<div class="csnStepTop">
  <div class="csnStepNum">2</div>
  <div class="csnStepTag">Extension</div>
</div>

<h3 class="csnStepTitle">Browser Extension (MV3)</h3>

<p class="csnStepDesc">

Real-time protection within the browser.

Key tasks:

- Navigation listener
- Scan request to API
- Pairing token flow
- Block page (`block.html`)
- Mode synchronization

</p>
